## How To Create A New Prospect

### Inputs needed
When you say "new prospective client," provide:

1. **Firm name** (e.g., "Tofer & Associates" → folder: `Tofer_and_Associates/`)
2. **Contact name + role** (person emailing us; e.g., "Jane Doe, Marketing Director")
3. **Their email to us** (paste verbatim — informs the wedge + opening bridge)
4. **How we got them** (cold outreach reply / referral / inbound / SB-37 angle / etc.)
5. **Known firm info** if any (Markets, size, decision-maker name, known vendors). Otherwise I research.

### What gets generated

Copy `_TEMPLATE_Firm_Name/` → `{{FIRM_NAME}}/` and populate:

| File | Populated from |
|---|---|
| `README.md` | Firm name, contact chain, their email, source |
| `01_Firm_Profile.md` | Public research — firm facts, verdicts, footprint |
| `02_Contact_Profile.md` | LinkedIn + any public profile on the contact |
| `03_Market_Analysis.md` | Their footprint × central Market pricing framework |
| `04_Pricing_Proposal.md` | Their Markets only — client-facing |
| `05_Strategy_Memo.md` | The tactical read: wedge, DM dynamic, what we'll NOT do |
| `06_Call_Script.md` | Socratic script customized to their situation |
| `07_Oracle_And_Negotiation.md` | Created 24h before call — oracle + buyer-tuned Voss |
| `08_Call_Notes.md` | Empty until after the call |
| `BRIEF_CHEATSHEET.md` | One-page scroll during call |
| `CALL_CHEATSHEET.md` | Full-page reference during/before call |

## Prospect → Client Transition

When a prospect signs:

1. **Move the entire folder** from `22_Outreach/07_Prospects/{{firm}}/` to `23_Clients/{{firm}}/`
2. Update any references that point at the old path (usually just the website TODO + any internal notes)
3. Add client-delivery artifacts to the moved folder: Service_Agreement, Audience/, In_Depth/ campaign work, Reporting/

The prospect files (01-08) become part of client history — useful context for onboarding, QBRs, and renewal conversations.

## Status Legend (for each prospect's `README.md`)

- **pre-outreach** — identified, no contact yet
- **cold-email-sent** — sequence started, no reply
- **discovery-scheduled** — reply received, call booked
- **discovery-complete** — first call held, follow-up in progress
- **proposal-sent** — materials delivered, awaiting DM
- **negotiation** — second call with DM in flight
- **signed** — move folder to `23_Clients/`
- **dormant** — paused, may revive (see `_STATUS.md`)
- **dead** — lost / closed (archive eventually)
