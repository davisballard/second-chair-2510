   - Laura: 263 (AZ pedestrian fatalities, 2024 ADOT Crash Facts)

Per Sharp + Indigo + Julian: **don't fragment Phase 1 learning.** Phase 2 introduces variants on top of validated winners — not 18+ new creatives flooding a single ad set. Target 2-3 new variants per validated persona, max 8-12 new units across Phase 2.

## Honest 60/30/10 reframe

The "60/30/10" shorthand that's surfaced in working notes is **directional language for thinking about lane balance**, not Julian's actual budget allocation. Reading the strategy file end-to-end:

- **60%-equivalent:** the proven hypothesis test = the 6 cinematic Phase 1 primaries. The lane WCTL is staking the campaign on.
- **30%-equivalent:** the emerging lane = DR copy-led + persona-specific Hero Number variants (Phase 1 refresh pool + Phase 2 expansion). Where new audience signal can be discovered.
- **10%-equivalent:** experimental = anything Phase 3+ outside the validated recipe (e.g., motion-still variants of DR units, Reels-eligible cuts of the cinematic primaries, etc.).

**Julian's actual budget allocation** per `05_Creative_Strategy_And_Ad_Set_Architecture.md`:

- **70% core** — the cinematic primary lane carrying the validated hypothesis test
- **20% flex reserve** — refresh-pool budget that can be deployed mid-flight if a Phase 1 unit underperforms or a Phase 2 candidate over-performs
- **10% buffer** — held against unexpected costs (audience-targeting waste, creative iteration, Saul-required compliance re-renders)

The phased Phase 1 / Phase 2 / Phase 3 hypothesis-validation gating is what actually controls media-spend allocation in WCTL — not a fixed split. The 60/30/10 frame is useful for *describing* the recipe to clients (Robert reads it cleanly); the 70/20/10 + Phase 1/2/3 gates are what *operate* it.
